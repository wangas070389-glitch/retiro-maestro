To **Achieve Functional Parity** for **Retiro Maestro** by **Integrating Legacy Prototype Logic**, enabling **Inverse Strategy Design** and **Fiscal Awareness** while adhering to **Protocol 23 (Legacy)** and **Protocol 31 (Adversarial)** precision.

## 3. Implementation Tactics
* **Inverse Design Engine (Custom Builder)**: A new calculation mode that accepts `target_pension` and iterates to find the minimum `daily_salary` and `years` required.
* **Fiscal Intelligence (ISR Layer)**: Integrating the UMA-based tax exemption logic (15 UMAs) to show "Real Money" (Net) vs "Paper Money" (Gross).
* **The "Escalera" Engine**: Mapping pension increments for every age from 60 to 65 in a single comparative view.
* **Employment Status Simulation**: Logic to freeze or grow weeks based on `isEmployed` status during wait periods.
* **Legal Document Template Service**: A service to generate the specific legal "Escritos" discovered in the prototype.

## 4. Key Metrics
* **Parity Score**: 100% of functional blocks from `prototype_retiromaestro73.txt` implemented in the modular architecture.
* **Inverse Precision**: Target pension achieved within +/- 1% margin.

## 5. Design Aesthetics (UX-009)
* **High-Fidelity Strategy Cards**: Utilizing "Narciso" glassmorphism to highlight the "Recommended" path with golden accents and deep shadows.
* **The "Escalera" Interaction**: A horizontal scroll/step component that visualizes pension growth by age (60 to 65) as a progressive ladder.
* **Actionable Document Previews**: Mini-rendered previews of the legal letters before generation to build user confidence.

## 6. Technical Architecture (Frontend)
*   **Zero-Layout Shift Policy**: The application shell must use **CSS Grid** (`grid-cols-[280px_1fr]`) instead of Flexbox to ensure rigid separation between the Sidebar and Main Content.
*   **Native Scrolling**: The body must allow native window scrolling (no `overflow-hidden` locks) to prevent "frozen" states on smaller viewports, while the Sidebar remains `sticky`.
*   **Viewport Integrity**: Top-level padding is minimized (`pt-2` to `pt-8`) to eliminate whitespace gaps and maximize information density.
