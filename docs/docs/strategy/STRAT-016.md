# STRAT-016: Report Sovereignty

## Vision
The "Report Sovereignty" initiative aims to transform the ephemeral screen experience into a durable, professional artifact. The user should be able to walk away with a **Tangible Asset**—a PDF report that serves as a roadmap for their retirement.

## Objectives
1.  **Professionalism**: The report must look like it came from a high-end actuarial firm, not a web page printout.
2.  **Portability**: Logic must execute client-side or server-side without reliance on browser print dialogs.
3.  **Clarity**: The layout must explain the "Why" and "How" of the pension calculation, not just the "What".

## Key Metrics
-   **Generation Speed**: < 2 seconds.
-   **Visual Fidelity**: Vector-based graphics and selectable text (no screenshots).
-   **Completeness**: Must include Scenario Comparison, Amortization Table, and Legal Citations.

## Interaction Model
-   **Trigger**: "Descargar Reporte PDF" button on Dashboard or Modal.
-   **Feedback**: Toast "Generando Reporte Soberano..." -> Automatic Download.
-   **Format**: PDF A4, Portrait.
