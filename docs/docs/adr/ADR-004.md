# ADR-004: Standardization of the Forensic Evidence Hash

## Status
Proposed

## Context
STRAT-004 requires a verifiable proof that a projection was generated using specific official anchors and formulas at a specific time. This "Evidence Hash" must be durable and resistant to manual tampering.

## Decision
Adopt a **Compound SHA-256 Semantic Hash** for all projection records. The hash will be calculated as:
`Hash = SHA256(Formula_ID | Anchor_Salt | Input_State | Timestamp)`

Key components:
1.  **Formula_ID**: Identifier for the deterministic logic version (e.g., `MATH-001v1.1`).
2.  **Anchor_Salt**: A cryptographic salt derived from the `legal-anchors.json` hash to ensure legal context unity.
3.  **Input_State**: Normalized JSON string of user inputs (Weeks, Salary, Family).
4.  **Local Fingerprint**: A non-PII browser/device identifier to link the record to the sovereign origin.

## North Star Attribute
**Auditability**

## Consequences
*   **Positive**: Projections can be independently verified by any auditor with the same `MATH-001` engine and `legal-anchors.json`.
*   **Negative**: Slight latency during hash generation on mobile devices.
*   **Maintenance**: Every legal update to anchors requires a version bump in the signing salt.
