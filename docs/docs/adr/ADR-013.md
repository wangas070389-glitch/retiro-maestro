# ADR-013: Trinity Module Architecture

## Status
Accepted

## Context
The application is currently structured as a single "Dashboard" page. The target prototype introduces distinct functional areas (Tools, Documents, Simulations) that would clutter a single view. To support the "Sovereign Suite" vision (`STRAT-013`), we need a scalable navigation and routing structure.

## Decision
We will restructure the application into a **Trinity Module Architecture**:

1.  **Route Separation**:
    -   `/dashboard` -> **Command Center** (Overview & Core Strategies)
    -   `/laboratory` -> **The Laboratory** (Tools: Tax, Retro, Simulators)
    -   `/authority` -> **The Authority** (Documents & Legal)

2.  **Shared Layout**:
    -   A persistent **Sidebar Navigation** (adapted from the prototype's menu) will provide access to these three modules.
    -   The `Layout` component will hold the `ToastContext` and `SessionProvider`.

3.  **Component Migration**:
    -   The current `InverseDesign` component will be moved to/integrated with the **Laboratory**.
    -   The `Escalera` logic will be centralized for use across Dashboard and Laboratory.

## Consequences
-   **Positive**: Improved code organization; better user experience (focused views); easier to scale new tools.
-   **Negative**: Requires refactoring existing `page.tsx` code into separate route components and setting up a new Layout structure.
