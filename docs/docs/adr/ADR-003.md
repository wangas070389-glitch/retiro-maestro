# ADR-003: Selection of Sovereign Ingestion Engine

## Status
Proposed

## Context
STRAT-003 requires extracting actuarial data from IMSS PDFs with perfect fidelity.
Constraints:
1.  **Privacy (Protocol 0)**: No PII should leave the local environment or sovereign boundary.
2.  **Accuracy (Protocol 17)**: Error rate must be negligible for numbers (Weeks, Salary).
3.  **Layout Diversity**: IMSS document layouts are inconsistent.

## Decision
Utilize a **Hybrid Ingestion Approach**:
1.  **PDF-Native Parsing**: Use `pdf-parse` for text-based PDFs to preserve character fidelity.
2.  **Local Heuristic OCR**: For scanned documents, use a local Tesseract.js instance within the sovereign browser context.
3.  **Protocol-Driven Sanitization**: Mandatory stripping of PII (Names, Addresses) before the data enters the simulation state.

## North Star Attribute
**Forensic Integrity**

## Consequences
*   **Positive**: Zero data leakage to 3rd party OCR APIs.
*   **Negative**: Higher local memory consumption during ingestion.
*   **Risk**: Potential for lower accuracy on low-quality manual scans compared to cloud-based LLM OCRs. Mitigation: Manual "Human-in-the-loop" confirmation for low-confidence scores.
