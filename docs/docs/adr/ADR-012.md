# ADR-012: Hybrid Theme Mandate

## Status
Accepted

## Context
The migration to Next.js introduced a "Full Dark Mode" default to match the "Sovereign" branding. However, this deviated from the proven "Narciso" prototype, which used a **Hybrid Theme** (Light Content Area / Dark Sidebar). Users perceive the Full Dark Mode as a regression in visual clarity and "premium" feel.

## Decision
We will enforce a **Hybrid Theme Architecture** for the Sovereign Dashboard:

1.  **Canvas Strategy**:
    -   **Main Content Area**: MUST be **Light Mode** (`bg-slate-50`). Representing "The Paper Record".
    -   **Sidebar / Navigation**: MUST be **Dark Mode** (`bg-slate-900`). Representing "The Machine / Control Plane".

2.  **Component Standards**:
    -   **Strategy Cards**: Must be **White** (`bg-white`) with subtle borders and shadows.
    -   **Typography**: Must use `Slate-900` for headings and `Slate-600` for body text in the light area to ensure maximum legibility.

3.  **No "Toggle"**: 
    -   We will **NOT** implement a user-toggleable Light/Dark switch at this stage. The "Sovereign" look is opinionated and curated.

## Consequences
-   **Positive**: Restores visual parity with the prototype; improves readability for data-heavy tables.
-   **Negative**: Requires refactoring existing `Dashboard` and components to remove hardcoded `text-white` classes.
