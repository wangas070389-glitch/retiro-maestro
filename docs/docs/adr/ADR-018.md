# ADR-018: Unified Reactive State for Dashboard Components

## Status
**ACCEPTED**

## Context
Multiple dashboard components (Main Form, Strategy Cards, Inverse Calculator) require access to the same `PensionInput` state. Currently, the main form uses local state/formData, while other components use a global store, leading to "State Drift" where cards don't reflect typed changes until a manual trigger.

## Decision
We will migrate the Dashboard input form to a fully controlled React model synchronized with `useSimulationStore`. The `StrategyCards` component will use `useMemo` to react to these store changes autonomously.

## Rationale
By making the store the primary state container, we ensure that as the user adjusts their retirement parameters, the "Laboratory" evolves in real-time. This provides a superior UX where the user can "see" the impact of adding 50 weeks or increasing their salary instantly across all potential strategies.

## Consequences
- **High Reactivity**: Instant updates to all cards.
- **Component Decoupling**: Cards no longer need to be "manually" refreshed by the parent form.
- **State Complexity**: Slightly higher overhead in the main form due to controlled inputs.
