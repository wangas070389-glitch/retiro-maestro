# ADR-014: Kinetic Interface & Mobile Sovereignty

## Status
ACCEPTED

## Context
Retiro Maestro has achieved functional parity with the prototype, but the user experience lacks the "premium" feel defined in the "Sovereign" aesthetic. The application feels static, and mobile responsiveness is functional but not optimized for touch. To achieve "Visual Sovereignty," we need to standardize motion and mobile interaction patterns.

## Decision
We will adopt a **"Kinetic First"** and **"Mobile Sovereign"** design philosophy, enforced by the following technical standards:

### 1. Motion Engine (`framer-motion`)
*   **Standard**: All state changes involving data (e.g., calculating pension) or layout (e.g., switching modules) must be animated.
*   **Library**: `framer-motion` will be the exclusive library for complex animations.
*   **Transitions**:
    *   **Page**: `AnimatePresence` wrapper for smooth cross-fades between routes.
    *   **Lists**: `LayoutGroup` for reordering or filtering lists (e.g., Strategy Cards).
    *   **Numbers**: Custom `CountUp` hook for all financial data changes.

### 2. Glassmorphism System
*   **Standard**: UI components will use a layered "Glass" aesthetic to create depth.
*   **Tokens**:
    *   `bg-glass`: `backdrop-filter: blur(12px)` + `bg-white/5` (Dark Mode).
    *   `border-glass`: `border-white/10`.
    *   `shadow-glow`: `box-shadow: 0 0 20px rgba(79, 70, 229, 0.15)` (Indigo glow).

### 3. Mobile Sovereignty
*   **Rule**: "Phone First, Desktop Scaled."
*   **Navigation**: The Sidebar must convert to a gesture-controlled **Drawer/Sheet** on screens < 768px.
*   **Inputs**: Font size must be >= 16px to prevent iOS zoom.
*   **Touch Targets**: Minimum 44x44px for all interactive elements.

## Consequences
*   **Positive**: massively improved perceived performance and "delight."
*   **Negative**: increased bundle size (~30kb for framer-motion) and development complexity.
*   **Mitigation**: Use lazy loading for heavy motion components and strict component reusability.
