# ADR-024: The 30.4 SAT Standard for ISR Calculations

## Status
Accepted

## Context
During the parity validation phase (Thelma case, PM-013), a persistent fraction-of-a-peso discrepancy appeared between the actuarial engine net pension results and the manual baseline model. 
We identified the root cause as the conversion factor from Daily UMA/Salary to Monthly. In some instances, the engine used `30` or `30.41` to represent a generic month.

## Decision
All fiscal and tax-related calculations within the `TaxEngine` and `PensionEngine` must utilize the strict **`30.4`** factor.
This multiplier is derived from the standard SAT rule (365 / 12 = 30.416, rounded down to 30.4 by local accounting standards for tax bracket estimations) and ensures perfect parity with externally certified manual calculations.

## Consequences
- **Positive**: Eradicates fractional errors across all lifetime projections (20+ years). Guarantees exact parity with legacy actuarial models.
- **Negative**: Requires strict oversight to ensure `30` or `30.416` is not reintroduced by standard generalized libraries. The value `30.4` is now treated as a hardcoded economic constant.
