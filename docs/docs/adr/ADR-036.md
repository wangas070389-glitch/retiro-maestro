# ADR-036: Monetization Consolidation & Legacy Tier Deprecation

## Status
**ACTIVE**

## Context
The project has experienced "Monetization Drift" where legacy tiers (`GOLD`, `SOVEREIGN`) and disparate paywall components coexist with the new B2C (`STRATEGY`, `DOSSIER`) and B2B (`STARTER`, `GROWTH`, `PRO`) models. This inconsistency creates conflicts in feature gating, trial guards, and administrative user management.

## Decision
1. **Absolute Tier Purge**: Systematic removal of all string literals and logic referencing `GOLD` or `SOVEREIGN` tiers.
2. **Definitive Hybrid Model Mapping**:
   - **B2C (Citizen)**: `FREE` -> `STRATEGY` -> `DOSSIER`.
   - **B2B (Advisor)**: `FREE` -> `STARTER` -> `GROWTH` -> `PRO`.
3. **Single-Source Enforcement**:
   - `lib/config/pricing.ts` is the absolute authority for feature-to-tier mapping.
   - `lib/auth/access-control.ts` and `lib/trial-guard.ts` must use the unified hybrid tiers for all bypass and gating logic.
4. **UI Consolidation**: `ProductPaywall.tsx` is designated as the **Only Definitive Paywall**. All other payment/upgrade UI elements are deprecated.
5. **Role-Tier Integrity**: Server-side validation in `monetization-actions.ts` to ensure users cannot acquire tiers that do not match their assigned `Role`.
