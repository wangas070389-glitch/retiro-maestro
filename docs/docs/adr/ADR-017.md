# ADR-017: Implementation of Inverse Actuarial Modeling

## Status
**ACCEPTED**

## Context
Standard pension calculators work forward: `Inputs -> Result`. Users frequently express intent in terms of the result: "I want a $50,000 pension." Manually finding the inputs (Salary, weeks) through trial and error in UI sliders is suboptimal for UX Sovereignty.

## Decision
We will implement an "Inverse Algorithm" that solves for the required Salary Daily Social Security (SBC) given a target pension amount and a projected age/weeks profile (defaulting to Age 65 and maximum weeks gain).

## Rationale
Since the Ley 73 formula is deterministic and linear with respect to the SBC (after applying the Cuantia brackets), we can algebraically invert the formula to find the required SBC without expensive iterative loops.

## Consequences
- **Instant Result**: The UI responds immediately to typing.
- **Accuracy**: Mathematical precision instead of numerical approximation.
- **Boundaries**: Clearly communicates the 25 UMA cap when a goal is mathematically impossible.
