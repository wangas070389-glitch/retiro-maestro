# ADR-023: SaaS Monetization & Tiering Strategy

## Status
Proposed (Implemented)

## Context
Retiro Maestro is evolving from a local utility to a cloud-native SaaS platform. To ensure long-term sustainability and fund continuous actuarial research, we need a monetization model that balances accessibility for individual workers with professional tools for consultants.

## Decision
We implemented a hierarchical **Account Tiering** system:

1.  **Tiers Definition**:
    - **FREE**: Basic simulations; in-memory results; standard PDF report.
    - **GOLD**: Persistent simulations; Forensic Dossier access; custom strategy design; Oracle real-time sync.
    - **SOVEREIGN**: All Gold features + Professional Legal Document Suite (M40, Baja Patrón) + Multi-device sync.
2.  **Monetization**: Simulated Stripe integration via a "Sovereign Upgrade" portal in the Authority and Dashboard pages.
3.  **Data Isolation**: Every simulation is bound to a `userId` at the database level, ensuring strict multi-tenant privacy while allowing for cloud-native convenience.

## Consequences
- **Positive**: Clear path to revenue; tiered value proposition; protection of high-stakes IP (forensic logic).
- **Negative**: Risk of "feature gate" frustration (mitigated by a generous Free tier behavior).
- **Compliance**: Adheres to LFPDPPP (Mexico Data Privacy) by ensuring user data is isolated and exportable.
