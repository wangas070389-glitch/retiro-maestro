# ADR-016: Selection of PDF Generation Engine

## Status
Accepted

## Context
We need to generate high-quality PDF reports for the user's pension strategy.
Options considered:
1.  **Browser Print (window.print())**: Cheap, but unprofessional formatting. Hard to control page breaks.
2.  **html2canvas + jspdf**: "Screenshot" based. Large file size, non-selectable text, pixelated on print.
3.  **@react-pdf/renderer**: Component-based, vector output, server/client compatible, high control.

## Decision
We will use **@react-pdf/renderer**.

## Rationale
-   **Component Model**: Reuses React mental model (View, Text, StyleSheet).
-   **Quality**: Generates true PDF vectors/text, ensuring crisp printing.
-   **Control**: Precise control over page breaks, headers, and footers which is critical for the "Amortization Table".

## Consequences
-   **Effort**: We must "re-implement" the UI components specifically for PDF (cannot just dump the HTML).
-   **Reward**: The result will be a premium, branded document suitable for a "Sovereign" application.
