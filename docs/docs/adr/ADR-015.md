# ADR-015: Visualization & Modal Architecture

## Context
Phase 10 requires displaying complex datasets (year-by-year financial projections) inside the main Dashboard without navigating away. We need a standard for Charts and Dialogs.

## Decisions

### 1. Visualization Library: `recharts`
-   **Decision**: Use `recharts`.
-   **Rationale**:
    -   Built for React (component-based).
    -   Lightweight enough for our needs.
    -   Excellent SVG animations which align with "Visual Sovereignty" (Kinetic UI).
    -   Easy to customize axis/tooltips.

### 2. UI Pattern: Radix UI Dialog (via Shadcn/ui)
-   **Decision**: Use a centralized `StrategyDetailModal` component triggered by state, rather than a new URL route.
-   **Rationale**:
    -   Keeps the "Dashboard" state alive (doesn't trigger a full page reload or layout shift).
    -   Allows quick comparison (Open "Rampa", Close, Open "Topado").
    -   Supports backdrop blur (Glassmorphism consistency).

### 3. Data Source: Client-Side Computation
-   **Decision**: The `PensionEngine` will expose a new method `getProjection(input, years)` that returns an array of objects.
-   **Rationale**:
    -   The input data is already on the client.
    -   Calculating 5-10 iterations of the pension formula is negligible for the client CPU (<5ms).
    -   Avoids unnecessary API round-trips.

## Consequences
-   Need to install `recharts` dependency.
-   Need to refactor `PensionEngine` to support "Time-Travel" calculations (calculating hypothetical future states).
