# ADR-011: Sovereign UX Standards

## Status
Accepted

## Context
Retiro Maestro's current UI uses standard browser alerts and basic inputs. This undermines the "Premium" and "Sovereign" trust model defined in `STRAT-005`. Users perceive `alert()` boxes as "cheap" or "insecure."

## Decision
We will enforce the **Narciso Design System** strictures for all user interactions:
1.  **Non-Blocking Feedback**: Replace `window.alert()` with a custom **Toast System** (Glassmorphic, animated).
2.  **Mobile-First**: All layouts must use Flexbox/Grid responsive patterns (Stack on Mobile, Sidebar on Desktop).
3.  **Active States**: All server actions must show a "Pulse" or "Skeleton" state, never a static frozen screen.

## Consequences
### Positive
*   **Trust**: High-polish UI signals "Professional Engineering."
*   **Retention**: Less friction during "Save" operations.
*   **Accessibility**: Toasts are screen-reader friendly (ARIA-live), unlike blocking alerts.

### Negative
*   **Effort**: Requires building custom components (Toast Context, Skeletons) instead of using native browser features.
*   **Bundle Size**: Slight increase in JS payload for animation libraries (Tailwind Animate).

## Compliance
*   **Audit**: Any PR introducing an `alert()` will be rejected by linting rules.
*   **Sovereignty**: The UI layer remains decoupled; the "Sovereign Record" (JSON) is valid regardless of the interface used to view it.
