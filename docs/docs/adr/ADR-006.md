# ADR-006: Self-Contained Forensic Bundle Format

## Status
Proposed

## Context
Standard projection signatures (STRAT-004) verify the *calculation* but do not prove the *context* (legal anchors like UMA/SMDF) used at the time. To ensure multi-decadal legal durability, we need a bundle format that includes all dependencies.

## Decision
We will implement a **Compound JSON Forensic Bundle (CJFB)**. This is a single JSON structure that embeds:
1. `meta`: Versioning and user-defined metadata.
2. `projections`: The array of signed evidence tuples.
3. `anchors_snapshot`: The exact `legal-anchors.json` used for these projections.
4. `digest`: A secondary SHA-256 hash of the entire bundle (excluding the digest field itself).

## Consequences
* **Positive**: Projections can be verified in the future even if the application's legal anchors have changed.
* **Positive**: Simple portability (single file export).
* **Negative**: Slightly larger file size (~10-20KB vs 1KB for a single signature).
