# ADR-032: 3-Zone Semantic Architecture for Authority View

## Status
Accepted

## Context
The previous "The Authority" panel was functionally robust but presented a high cognitive load, effectively serving as a generic "utilities panel." The layout gave equal visual weight to disparate functions (PDF generation, cryptographic auditing, and SaaS upgrades), diffusing User Experience and minimizing the perceived value of the Sovereign Dossier.

## Decision
The view has been entirely restructured and renamed to **"Panel Legal & Auditoría"** using a 3-Zone Semantic Architecture:
1. **Zona 1: Flujo Operativo Inmediato**: Operational friction-reducers (Generador de Escritos, Calendario Inteligente). 
2. **Zona 2: Seguridad y Auditoría Criptográfica**: High-trust operations (Dossier Soberano verification, Oracle Pulse synchronization).
3. **Zona 3: Escalamiento Institucional**: The conversion path (Upgrade Premium).

## Consequences
- **Positive**: Introduces a narrative flow (Create -> Accumulate Trust -> Upgrade). Eliminates the "utility panel" feel, elevating the system's perceived authority and legal validity. Directly impacts SaaS conversion metrics.
- **Negative**: The page is vertically longer and requires more sophisticated responsive handling to maintain visual hierarchy across the three zones.
