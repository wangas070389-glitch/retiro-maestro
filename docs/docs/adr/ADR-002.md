# ADR-002: Simulation State Management for Digital Twin

## Status
Proposed

## Context
The "Retiro Maestro" Digital Twin platform (STRAT-002) requires the ability to compare multiple retirement scenarios (A/B testing) in real-time. This involves handling high-frequency updates from UI sliders (Salary, Weeks, Age) and reconciling these inputs with the deterministic `PensionEngine` (MATH-001).

Current challenges:
1.  **A/B Comparison**: Users must be able to switch between "Current Path" and "Optimized Path" without state loss.
2.  **Concurrency**: Multiple simulation runs might trigger as the user adjusts sliders.
3.  **Data Sovereignty**: Scenario state must be serializable for export (Protocol 23).

## Decision
Adopt **Redux-style Immutable State Trees with Action-driven Transitions** for the simulation layer. Implementation will use **Zustand** with an immutable middleware for lightweight, high-performance state management that satisfies Protocol 13 (Signed Intent).

## North Star Attribute
**Reconcilability**

## Options Considered
1.  **Vanilla React State**: Fast but difficult to share across complex dashboard components (Graphs, Tables, Sidebar).
2.  **Zustand (Immutable)**: Selected for its minimal boilerplate and robust support for transient state (sliders).
3.  **Redux Toolkit**: Too heavy for the current scale, though the mental model is preserved.

## Consequences
*   **Positive**: Easy "Undo/Redo" for simulation scenarios. Clear audit trail of what inputs led to what result.
*   **Negative**: Slight overhead in defining actions for every slider interaction.
*   **Risk**: Potential state-ui desync if not carefully implemented with high-probability re-renders.
