# ADR 027: Inmutabilidad Reactiva del Sello Forense (Forensic Provenance)

## Contexto
El simulador `Retiro Maestro` permite la carga del documento origen (Constancia de Semanas Cotizadas) extrayendo parámetros vía `pdfjs-dist` + heurística en `HeuristicParser`. Sin embargo, existía el riesgo arquitectónico de que el usuario modificara los valores extraídos a mano antes de calcular, resultando en un expediente o acta PDF donde los datos presentados como "extraídos del documento oficial" fueran en realidad apócrifos.

## Decisión
Se ha decidido implementar un **Sello Forense Criptológico y Reactivo** (Forensic Provenance Seal) que reside en la memoria volátil del cliente base (`useSimulationStore`). 

1. **Minting (Acuñado):** El Uploader emite un `CertifiedDossier` en el Store solamente si el score heurístico y las validaciones de `SentinelAuditor` son superiores al umbral de confianza.
2. **Candado Visual:** La UI presenta el documento base bloqueando visualmente la edición.
3. **Pérdida por Fricción (Mutación):** Si el asesor muta manual y deliberadamente los campos `Semanas`, `Salario Promedio`, o `Edad` usando el teclado numérico / incrementos de la forma de react hooks, se ejecuta un *Intercept* puro que destruye el Sello Forense (seteando `certified_dossier` a nulo).
4. **Acoplamiento Directo al Impreso (PDF):** El acta que se descarga del cliente inyecta la franja de "Autenticidad Criptográfica" leyendo directamente de esta bóveda.

## Consecuencias Positivas
- **Imposibilidad de Falsificación Inadvertida:** El asesor no puede entregar un acta oficial de "100% de Confianza" con datos cuchareados.
- **Rápida validación:** El PDF inercial pasa de ser una *estimación* a ser un *documento vinculante y verificado localmente*.
- **Cero latencia de DB:** Al residir en el Zustand store durante la sesión, no requiere un backend para autentificar el documento.

## Consecuencias Negativas
- Aumenta la complejidad del acoplamiento entre la vista del Dashboard (formulario libre) y el objeto de estado (que ahora actúa como un guardia de inmutabilidad vigilando el payload de escritura).
