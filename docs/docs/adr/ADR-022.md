# ADR-022: Economic Oracle Persistence & Integrity

## Status
Proposed (Implemented)

## Context
Projecting pensions for 20+ years requires precise economic data (UMA, INPC). Previously, these were hardcoded in `legal-anchors.json`. To achieve "Real-Time Sovereignty," the system must ingest this data from official sources (DOF/INEGI) and persist it without compromising forensic integrity.

## Decision
We implemented a dedicated **Economic Oracle** layer:

1.  **Persistence**: Data is saved to the `EconomicAnchor` table with a versioning strategy (year + source).
2.  **Integrity**: Every ingested value is tagged with a `signature` (simulated cryptographic seal) and `is_verified` flag.
3.  **Adversarial Veto (P31)**: The system automatically blocks any UMA/INPC update that deviates more than 15% from the previous value, preventing "Oracle Poisoning" attacks.
4.  **Simulation Binding**: When a simulation is saved, it now captures a snapshot of the active `EconomicAnchor` to ensure that years later, the projection can be re-verified against the exact economic context of its creation.

## Consequences
- **Positive**: Automated legal accuracy; reduced maintenance; increased user trust.
- **Forensic**: Past simulations remain valid even if the UMA changes, as the "context snapshot" is preserved in the integrity hash.
