# ADR-030: Generación de PDF en el Cliente con React-PDF

## 1. Contexto y Problema
En el marco de **STRAT-028**, la plataforma `retiro-maestro` requiere generar reportes en PDF detallados y visualmente precisos para entregar a los clientes finales.
Las opciones tradicionales para generación de PDFs en aplicaciones web suelen depender de servicios de Backend (como Puppeteer para renderizar HTML a PDF) o servicios de terceros. Esto viola nuestro principio de soberanía y simplicidad de infraestructura, creando cuellos de botella y posibles fallos si la red falla o el servidor en la nube no responde.

## 2. Decisión Arquitectónica
Se adopta **`@react-pdf/renderer`** como el motor exclusivo de generación de PDFs.

### Justificación
- **Renderizado Reactivo**: Permite construir el layout del PDF usando componentes de React estilizados (`<Document>`, `<Page>`, `<View>`, `<Text>`), manteniendo la coherencia mental con el resto de la base de código Next.js.
- **Client-Side Generation**: La librería es capaz de computar y generar el Blob del PDF enteramente en el navegador del Agente mediante WebAssembly/JS puro, cumpliendo la meta de "cero servidor backend" en la operación crítica.
- **Tipografía y Estilo**: Permite registrar y empaquetar fuentes propias (como Inter o Roboto) para garantizar que la estética sea idéntica sin importar la máquina en la que corra el reporte.

## 3. Consecuencias
- **Positivo**: Generación determinista, instantánea y offline-capable de reportes post-simulación.
- **Negativo**: `@react-pdf/renderer` no utiliza CSS estándar ni Tailwind; sus componentes tienen una API de estilo similar a React Native (atributos `StyleSheet.create`), lo que requerirá traducir nuestros tokens de diseño (colores, espaciados de Narciso) a sus especificaciones nativas en el módulo del reporte.
