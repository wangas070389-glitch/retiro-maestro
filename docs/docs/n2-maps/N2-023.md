# Sovereign Key Rotation State Machine (N2-023)

## Flow Overview
This diagram maps the logical execution flow of the `updatePasswordAction`. It enforces a strict authorization gate before allowing a user to rotate their cryptographic key.

```mermaid
stateDiagram-v2
    [*] --> SessionVerification

    state SessionVerification {
        CheckJWT: auth()
        ValidJWT: Extracted User.ID
        InvalidJWT: Throw Unauthorized
        
        CheckJWT --> ValidJWT
        CheckJWT --> InvalidJWT
    }

    SessionVerification --> LoadCurrentIdentity: Valid
    SessionVerification --> [*]: Invalid

    state LoadCurrentIdentity {
        PrismaSelect: db.user.findUnique
        HashFound: Retrieved Hash
        
        PrismaSelect --> HashFound
    }

    LoadCurrentIdentity --> CryptographicCheck

    state CryptographicCheck {
        BcryptCompare: bcrypt.compare(provided, db_hash)
        Match: Clearance Granted
        Mismatch: Throw "Contraseña Actual Incorrecta"
        
        BcryptCompare --> Match
        BcryptCompare --> Mismatch
    }

    CryptographicCheck --> KeyRotation: Valid Match
    CryptographicCheck --> [*]: Invalid Match

    state KeyRotation {
        HashNew: bcrypt.hash(newPassword, 10)
        PrismaUpdate: db.user.update(password: HashNew)
        
        HashNew --> PrismaUpdate
    }

    KeyRotation --> [*]: Success
```
