# Advisor Identity Impersonation Matrix (N2-024)

## Flow Overview
This state diagram models the logic required for an Advisor to view and mutate a Client's data without exposing other clients. It maps the architectural necessity of a "Delegated Session" context.

```mermaid
stateDiagram-v2
    [*] --> Authentication

    state Authentication {
        NextAuth: auth()
        RoleCheck: Is Role == ADVISOR?
        
        NextAuth --> RoleCheck
    }

    Authentication --> CRM_Dashboard: True
    Authentication --> [*]: False (Standard User)

    state CRM_Dashboard {
        FetchClients: db.user.findMany(where: {advisorId: session.id})
        RenderList: Display assigned users
        SelectClient: Advisor clicks a Client row
        
        FetchClients --> RenderList
        RenderList --> SelectClient
    }

    CRM_Dashboard --> Authority_Tunnel

    state Authority_Tunnel {
        ContextSwitch: Set ActiveClientContext = targetUserId
        VerifyLink: Validate targetUserId.advisorId == session.id
        
        ContextSwitch --> VerifyLink
    }

    Authority_Tunnel --> Simulation_Engine: Valid Link
    Authority_Tunnel --> Error_Forbidden: Invalid Link

    state Simulation_Engine {
        LoadDossier: Fetch strategies where userId == ActiveClientContext
        CalculateM40: Run projections using Client's base logic
        SaveStrategy: db.strategy.create(userId: ActiveClientContext)
        
        LoadDossier --> CalculateM40
        CalculateM40 --> SaveStrategy
    }

    Simulation_Engine --> [*]: Mission Accomplished
```
