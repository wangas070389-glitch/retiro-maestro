# N2-021: System Interaction Map - Intelligent Payment Calendar

Este mapa N2 ilustra las interfaces de datos estáticas y dinámicas dentro del módulo de Rendición de Calendarios (Authority UI). 

El objetivo es asegurar el flujo de solo-lectura desde las simulaciones seguras (Sovereign Dossier) hasta el UI predictivo, sin mutar datos de estado subyacentes.

```mermaid
graph TD
    subgraph "Core Data (Read-Only State)"
        B[Simulación en DB (Dossier)] -->|SBC Diario (S)| C(Payment Calendar Manager)
    end
    
    subgraph "Logic Layer (Compute)"
        C -->|S, Y_n, M_n| D{m40-calculator.ts}
        D -->|Busca Tasa de Salto| E[Diccionario Progresivo Ley 73]
        D -->|Aritmética de Tiempo| F[Resolutor de Fechas Gregorianas]
    end
    
    subgraph "User Interface Layer (Presentation)"
        D -->|Array Object P_m| G[Authority/page.tsx UI]
        G -->|Visual| H[Alertas de Inflación / Tasa UI]
        G -->|Visual| I[Tarjetas Mensuales con Estado (Pasado/Próximo)]
    end
```

## Flujos de Error Previstos
- **Null Reference:** Si `B` (Simulación) no existe, `C` aborta y manda señal a `G` para activar la "Vista Vacía" incitando a simular primero.
- **Out of Bounds Time:** Si el algoritmo intenta ir más allá de 2030, la lógica saturará en $18.80\%$.
