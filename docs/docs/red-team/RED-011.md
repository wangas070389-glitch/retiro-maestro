# RED-011: UX Vulnerability Analysis (Sovereign Interface)

## 1. Attack Vector: "Illusion of Authority"
*   **Hypothesis**: A polished UI (Narciso) may lead users to blindly trust incorrect calculations.
*   **Mitigation**: The "Signature Hash" must always be visible. Even if the UI is beautiful, the mathematical proof (JSON hash) must be the ultimate source of truth, displayed in monospace font (`font-mono`) to signal "Raw Data".

## 2. Attack Vector: "Mobile truncation"
*   **Hypothesis**: On small screens, vital warnings (e.g., "Meta Inalcanzable") might be pushed "below the fold" or truncated.
*   **Mitigation**: Critical alerts must use `position: sticky` or appear at the top of the computation stack to ensure visibility regardless of scroll position.

## 3. Attack Vector: "Animation Fatigue"
*   **Hypothesis**: The "Pulse/Skeleton" animations might cause motion sickness or annoyance for power users running hundreds of simulations.
*   **Mitigation**: Adhere to `prefers-reduced-motion` media queries. If the user OS requests reduced motion, disable the Pulse and use a simple static "Loading..." indicator.

## 4. Attack Vector: "Toast Spam"
*   **Hypothesis**: Rapid-fire calculations could flood the screen with toast notifications.
*   **Mitigation**: Implement a "Debounce" or "Replace" policy. Only show the *latest* result status; dismiss previous toasts immediately upon a new action.
