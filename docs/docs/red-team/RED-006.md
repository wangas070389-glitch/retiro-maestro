# RED-006: Forensic Sovereignty & Resilience (Red Team Analysis)

## 1. Attack Surface: Forensic Durability
The primary target is the **Compound Forensic Bundle**. An attacker (or a malicious lawyer) aims to invalidate the evidence by proving it is incomplete or manipulatable.

## 2. Theoretical Attack Vectors
| Vector | Description | Mitigation Strategy |
| :--- | :--- | :--- |
| **Anchor Desync** | Attacker modifies the `anchors_snapshot.json` within the bundle to change the outcome while keeping a "valid" signature. | Bundle-level Master Hash (ADR-006) must include the anchor snapshot. |
| **Time-Travel Collusion** | Manipulating the timestamp to claim a projection was made under a different legal regime. | Cryptographic chaining of previous bundle hashes. |
| **Schema Fragmentation** | The app evolves to a state where it can no longer parse older bundles, creating "Data Purgatory." | Protocol 23 (Legacy) mandatory human-readable Manifest.txt backup. |
| **Key Erasure** | The user loses the local "Private Key" if we move to asymmetric signing. | Maintain SHA-256 Compound Semantic Hashes which depend on data, not just secrets. |

## 3. Lindy-Proof Audit
* **Question**: Will this data be readable in 2046?
* **Risk**: JSON is standard, but the *semantic meaning* of "Weeks" might change.
* **Protocol**: Protocol 23 requires embedding the *definition of terms* (Formula Manifest) inside the bundle.

## 4. Failure Mode and Effects Analysis (FMEA)
* **Failure**: Bundle file corruption (1 bit flip).
* **Effect**: Verification fails; legal evidence is tossed in court.
* **Criticality**: High. 
* **Mitigation**: Implement Reed-Solomon error correction or simple redundant storage prompts.
