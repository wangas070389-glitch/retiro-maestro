# Profile Management Threat Vectors (RED-020)

## Attack Surface Analysis
The introduction of user-driven profile modifications (`/profile`) opens several vectors for abuse, specifically concerning account takeover and data corruption.

### 1. Insecure Direct Object Reference (IDOR)
*   **Vector:** A malicious actor intercepts the network request and attempts to modify the `userId` payload to change the email or password of a *different* user (e.g., an Admin).
*   **Mitigation:** The Server Actions (`updateProfileInfoAction`, `updatePasswordAction`) do **not** accept a `userId` parameter from the client. They strictly extract the `userId` from the cryptographically signed JWT via `await auth()`. 

### 2. Privilege Escalation via Profile Update
*   **Vector:** An attacker intercepts the user profile JSON payload and injects `role: "ADMIN"` or `tier: "SOVEREIGN"` into the request, hoping the backend blindly merges the object into Prisma.
*   **Mitigation:** The Prisma `update` query is rigidly structured to exclusively map the explicitly requested fields (`name`, `email` or `password`). Unknown fields are naturally dropped by TypeScript and Prisma schema validations.

### 3. Password Reset Hijacking
*   **Vector:** An attacker gains access to an unlocked computer and attempts to change the victim's password to lock them out.
*   **Mitigation:** The `updatePasswordAction` strictly mandates the `currentPassword` payload. It reads the target user's existing Bcrypt hash from the database and runs `bcrypt.compare`. If this check fails, the action immediately aborts, preventing opportunistic lockouts.

### 4. Email Collision (Denial of Service)
*   **Vector:** An attacker attempts to change their email to match an existing user's email, potentially causing unique constraint exceptions in the database and crashing the server action.
*   **Mitigation:** The `updateProfileInfoAction` must gracefully catch the Prisma `P2002` (Unique constraint failed) error and return a sanitized, localized error message to the client ("El correo ya estĂˇ en uso") without crashing the process.
