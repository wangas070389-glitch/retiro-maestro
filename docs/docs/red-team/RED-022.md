# RED-022: Zod Schema Evolution and Backwards Compatibility Validation

This document outlines the testing protocols, edge cases, and validations applied to ensure the system is immune to runtime crashes caused by database schema evolution or data shape drift.

## 1. The Parity Breach (Post-Mortem 016)

During the deployment of family assignments (+15% spouse, +10% children, +10% parents), the input validation schema was updated. The three new fields were added to the Zod parser as strict types:
* `has_wife`: `z.boolean()`
* `children_count`: `z.number()`
* `dependent_parents_count`: `z.number()`

This created an immediate validation failure when users loaded older simulation entries from the database, because historical records did not have these keys. safeParse failed, locking users out of their saved simulations.

## 2. Red-Team Mitigations

### 2.1 Schema Defaulting
We resolved this vulnerability by modifying `z.boolean()` and `z.number()` to include `.default()` fallbacks:
```typescript
const PensionSchema = z.object({
    // ...
    has_wife: z.boolean().default(false),
    children_count: z.number().min(0).max(10).default(0),
    dependent_parents_count: z.number().min(0).max(2).default(0),
    // ...
});
```

### 2.2 Validation Tests
To prevent regression, the following validation test cases were integrated:
1. **Empty / Minimal Object Input**: Pass an object containing only `{ weeks: 1250, salary_prom: 500, age: 60 }` to `calculatePensionAction`. Verify that it parses successfully and defaults the family allowances to `false`, `0`, and `0` respectively.
2. **Invalid Numbers Safeguard**: Pass string numbers (e.g. `'10'`) and verify that they are coerced or rejected safely.
3. **Out-of-Bounds Checks**: Verify that `children_count > 10` or `dependent_parents_count > 2` triggers z.error instead of running calculations.
4. **Vigencia Guard Check**: Confirm that if `is_ongoing_work` is false, `last_termination_date` must be present and valid.

## 3. Authority Verification Ledger

All 24 actuarial unit tests compile and run under the latest Zod schema, ensuring:
- Rights conservation active/expired logic is correct.
- Capped SBC calculations do not exceed 25 UMAs.
- Rounding rules for Ley 73 surplus semesters conform to the 2013 Reference Manual.
- Production build compiles with zero Next.js type or lint errors.
