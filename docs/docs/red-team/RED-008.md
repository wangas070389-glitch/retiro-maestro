# RED-008: Adversarial Validation (Red Team Analysis)

## 1. Attack Surface: Actuarial Integrity
The target is the **Precision and Logic** of the pension engine. An attacker (or corrupted input) aims to bypass validation and generate a "Legitimate-looking but False" projection.

## 2. Theoretical Attack Vectors
| Vector | Description | Mitigation Strategy |
| :--- | :--- | :--- |
| **Coefficient Overflow** | Inputting 9,999 weeks to see if the SBC calculation overflows or results in NaN. | Strict type-enforcement and `Math.min(weeks, 2750)` logic. |
| **Logic Flip** | Using an age of 14 (legal min) with 50 years of contributions (temporal impossibility). | `SentinelAuditor` temporal consistency bridge (STRAT-005). |
| **Rounding Death** | Repeatedly simulating increments of 0.01 to see if tax-rounding errors accumulate. | Unit-testing with high-precision decimal libraries for verification. |
| **Memory Exhaustion** | Flooding the Ingestion UI with 1GB "PDFs" (actually random data). | File-size boundaries and local-first stream-parsing timeouts. |

## 3. Lindy-Proof Audit
* **Question**: Does the engine hold up if the UMA value jumps by 20% in one year?
* **Risk**: High volatility in legal anchors could break hard-coded assumptions about "Expected" ranges.
* **Protocol**: Protocol 17 requires the `SentinelAuditor` to pull dynamic "Sanity Belts" from the `legal-anchors.json`.

## 4. FMEA (Failure Mode and Effects Analysis)
* **Failure**: Property-based test finds a case where $P_{m40} < P_{base}$.
* **Effect**: Foundational logic error discovered.
* **Criticality**: Maximum.
* **Mitigation**: Immediate dashboard lock and "Logic Recall" protocol implementation.
