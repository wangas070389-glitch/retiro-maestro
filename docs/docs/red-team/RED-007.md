# RED-007: Dynamic ROI Optimization (Red Team Analysis)

## 1. Attack Surface: Financial Projections
The primary target is the **ROI Calculation Logic**. An attacker (or logic error) aims to "Lure" the user into a bad investment by overstating returns or understating costs.

## 2. Theoretical Attack Vectors
| Vector | Description | Mitigation Strategy |
| :--- | :--- | :--- |
| **Inflation Denial** | Modeling zero inflation to make future payments seem more valuable than they are. | Mandatory INPC (Inflation) indexing toggle. |
| **Breakeven Hallucination** | Ignoring the "Cost of Capital" (Interest rates) for the investment months. | Add "Opportunity Cost" comparison (e.g. versus 10% CETES). |
| **Survival Bias** | Assuming 100% survival to age 100 to inflate the "Lifetime Delta". | Use official CONAPO Life Expectancy tables (capped at age 85). |
| **Tax Shadowing** | Ignoring the tax exemption status of pensions, leading to incorrect net income comparisons. | Explicitly state "Pre-Tax" or "After-Tax" status in results. |

## 3. Lindy-Proof Audit
* **Question**: Does the ROI math account for the yearly cost increases in Modalidad 40?
* **Risk**: The 2020 Reform significantly increases the cost of M40 every year until 2030.
* **Protocol**: The `ROIComputer` must use the graduated rate table (11.166% -> 18.8%).

## 4. FMEA (Failure Mode and Effects Analysis)
* **Failure**: Overestimating SBC (Salario Base de Cotización) due to incorrect UMA ceiling.
* **Effect**: The user over-invests but gets capped by IMSS, losing capital.
* **Criticality**: Critical.
* **Mitigation**: Sentinel-enforced 25 UMA hard-cap on all ROI inputs.
